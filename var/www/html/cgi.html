CGI Hello Generator

Interact with get_hello.py. This script reads GET or POST parameter inputs (first_name & last_name) parsed by the CGI gateway and prints dynamic greetings.

How standard form submission works:

Clicking "Standard Form" will instruct the browser to make a traditional GET navigation request. The server launches the python subprocess, pipes the parsed query string into the script, captures stdout, and streams the finished page directly back into a new tab.

CGI Math Engine

Interact with calc.py. It accepts query fields f_num (first operand), oper (arithmetic symbol), and s_num (second operand), performs server-side math, and outputs parsed C++ / Python values.

Dynamic Validation & Limits:

The backend calc.py handles security validation by ensuring the inputs are strictly digits and oper is a valid character. It also enforces a maximum limit of 10,000 to prevent CPU exhaustion on exponential operations, throwing error pages if violated!

CGI Environment Variables

Interact with env.py. Running this script reveals the comprehensive environment array setup by your C++ webserv server. These meta-variables provide essential CGI details like REQUEST_METHOD, PATH_INFO, CONTENT_TYPE, and HTTP client headers.

The CGI Protocol Bridge:

CGI (Common Gateway Interface) passes request information to script files by using Environment Variables. For example, the query string is written directly into os.environ['QUERY_STRING'], and client request headers are prefixed with HTTP_ and made uppercase.

CGI Cookie Manager

Interact with cookie handlers set_cookie.py, get_cookie.py, and cookie.py. This sandbox allows you to write custom key-values, and showcase how your CGI pipeline parses, stores, and sends HTTP cookies dynamically.

Browser Cookie Storage

Click "List All Cookies" to fetch cookies from active session!

How HTTP Cookie flow works:

When you set a cookie, the Python CGI output includes a Set-Cookie header which your Webserv transmits back to the browser. The browser stores this value. In all subsequent requests, the browser adds a Cookie header, which your Webserv feeds into the HTTP_COOKIE environment variable for the script to load!

CGI Multi-Part File Upload

Interact with upload.py. This showcases processing a full multi-part POST file upload via CGI. The script processes the raw HTTP stdin, parses boundary files, and writes the output payload into the server's /tmp directory.

Drop a file here, or browse to upload via Python CGI

CGI POST vs C++ native POST:

Unlike standard C++ POST routing (which is handled natively by the server logic in /uploads), CGI file uploads route the POST request body into the Python process's **standard input (stdin)**. Python reads the bytes, respects CONTENT_LENGTH, and handles the multipart boundaries manually!
